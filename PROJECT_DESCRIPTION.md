# Hello World DApp

## 產品概述

一個極簡的 Sui DApp，讓使用者能：
1. **建立**一則儲存於鏈上的問候訊息
2. **檢視**自己已建立的問候
3. **更新**該問候內容

單頁應用，流程清晰、操作直覺，以「建立 → 檢視/更新」為主軸，不堆砌多餘功能。

---

## 設計與風格（必讀）

生成的前端必須**優雅、有辨識度**，避免 generic「AI 感」介面。請遵循以下原則：

### 視覺
- **主題**：深色為主，但不要純黑一片。使用深灰/墨色背景（例如 `#0f0f12`、`#16161a`），搭配少量高對比留白或淺色區塊區分層級。
- **色彩**：一套主色 + 一個點綴色即可。主色用於按鈕、連結、重點資訊；點綴色用於成功/焦點狀態。避免紫白配或過度飽和的漸層。
- **字體**：選用有特色的字體組合（例如一個用於標題、一個用於內文），並從 Google Fonts 等引入。不要使用 Inter、Roboto、Arial、Space Grotesk 等過度常見的預設字體。
- **留白與層級**：按鈕、表單、卡片之間留足間距；重要操作（建立/更新）視覺權重高於次要操作；錯誤/成功訊息清楚但不搶戲。

### 互動與動線
- **動效**：過場或回饋可採用輕量 CSS 過渡（例如 200–300ms），避免花俏動畫。按鈕 hover、表單 focus 狀態要明確。
- **狀態**：未連線錢包時，友善提示連線並保持版面完整；連線後才解鎖建立/更新。載入中、交易 pending 時要有明確狀態指示（spinner 或文字），避免畫面「卡住」感。
- **表單**：輸入問候內容的欄位要顯眼、好填；送出前可做基本驗證（非空、長度等）。成功寫入鏈上後，介面應即時反映新內容（或明確提示重新查詢）。

### 技術約束
- 使用 **@mysten/dapp-kit-react** 與 **@mysten/sui** 最新版本；錢包連線、簽名與交易流程符合 dApp Kit 建議模式。
- 一頁式設計，不另做多頁路由；所有功能（連線、建立、檢視、更新）在同一視圖內完成，可依狀態切換區塊或表單顯示。

---

## 總結給生成器的要點

- **功能**：建立鏈上問候 → 檢視我的問候 → 更新問候；單頁完成。
- **風格**：深色、優雅、有辨識度的字體與配色，留白與層級清楚，輕量動效。
- **體驗**：連線狀態明確、載入與錯誤有回饋、表單簡潔好操作。
